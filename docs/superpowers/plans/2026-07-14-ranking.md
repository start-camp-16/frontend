# 뭐할구 프론트엔드 2단계: 장소 랭킹 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 구·카테고리 선택, URL 상태, 원본 순위, 페이지네이션과 복구 가능한 상태를 갖춘 `/` 랭킹을 구현한다.

**Architecture:** API adapter, query state, DOM view, page controller를 분리한다. URL이 선택·페이지의 단일 기준이며 새 요청은 이전 요청을 취소한다.

**Tech Stack:** 기존 Vite/Vanilla JS 기반, Vitest/jsdom, Playwright

## Global Constraints

- 카테고리와 구는 API에서 로드하고 프론트에서 순위·순서를 계산하지 않는다.
- 기본 `size`는 20이며 URL 쿼리는 `district`, `category`, `page`만 사용한다.
- 이미지가 없거나 실패하면 공통 SVG를 표시한다.
- 360px에서 동일 기능과 키보드 조작을 제공한다.
