# 뭐할구 프론트엔드 코스 생성·공유 설계

## 목적

사용자가 서울의 구, 관심 카테고리, 방문 장소 수를 선택해 코스 초안을 받고, 장소와 순서를 편집한 뒤 익명으로 저장·공유할 수 있게 한다. 공유된 코스는 비밀번호로만 수정·삭제한다.

백엔드가 추천 순서를 정할 때 사용하는 직선거리와 총 직선거리는 프론트엔드에 표시하지 않는다. 거리 값은 화면 문구, 요약, 접근성 텍스트에도 노출하지 않는다.

## 범위

### 포함

- 공통 상단 메뉴를 `홈 / 코스 / 게시판`으로 확장
- `/courses`에서 추천 조건 입력, 초안 생성, 장소 추가·삭제·순서 변경, 익명 저장
- `/courses/{public_id}`에서 공개 조회, 링크 복사, 비밀번호 기반 수정·삭제
- 기존 메타데이터 API를 이용한 구·카테고리 선택
- 기존 랭킹 API를 이용한 수동 장소 추가
- 추천과 코스 CRUD 오류 상태 및 재시도
- 키보드 탐색, 모바일 레이아웃, 전체 사용자 흐름 자동화 테스트

### 제외

- 직선거리, 총 거리, 실제 경로, 예상 이동시간 표시
- 지도와 경로선
- 공개 코스 목록, 검색, 좋아요, 조회수
- 로그인, 사용자별 코스 목록, 공동 편집, 수정 이력
- 새로고침 이후 저장되지 않은 초안 복구

## 라우트와 내비게이션

- `/`: 기존 장소 랭킹 홈
- `/courses`: 코스 생성 작업대
- `/courses/{public_id}`: 공개 코스 조회 및 소유자 편집
- `/posts`: 기존 게시판

공통 내비게이션은 `홈 / 코스 / 게시판` 순서로 표시한다. 공개 ID는 UUID4의 하이픈 없는 32자 문자열이므로 라우터는 정확히 32자의 16진수 소문자(`[0-9a-f]{32}`)로 이루어진 코스 경로만 허용한다.

## 코스 생성 작업대

`/courses`는 데스크톱에서 조건 패널과 코스 타임라인을 나란히 배치하고, 모바일에서는 위아래로 쌓는 한 화면 작업대다.

### 조건 패널

- 구 1개 선택
- 카테고리 1~3개 선택
- 방문 장소 수 3~5개 선택
- 선택한 카테고리 수가 장소 수보다 많지 않도록 즉시 검증
- 유효한 조건에서만 `코스 초안 만들기` 활성화

페이지 진입 시 구와 카테고리 메타데이터를 병렬로 불러온다. 어느 하나라도 실패하면 조건 입력을 비활성화하고 동일 위치에서 다시 시도할 수 있게 한다.

### 코스 타임라인

추천 성공 시 API가 반환한 `stops` 배열 순서를 그대로 초안으로 사용한다. 장소 카드는 순번, 이미지, 장소명, 카테고리, 주소를 표시한다. `distance_from_previous_meters`와 `total_straight_line_distance_meters`는 렌더링하지 않는다.

각 장소는 `위로`, `아래로`, `삭제` 동작을 제공한다. 드래그만으로 순서를 바꾸게 하지 않아 키보드와 터치 환경에서 같은 기능을 보장한다. 첫 장소의 `위로`와 마지막 장소의 `아래로`는 비활성화한다.

초안은 서로 다른 장소 3~5개를 유지해야 한다. 삭제 결과가 3개 미만이 되는 동작은 실행하지 않고 이유를 안내한다. 5개가 되면 장소 추가를 비활성화한다.

### 장소 추가

장소 추가 패널에서 구와 카테고리를 선택하고 기존 `GET /api/rankings`를 조회한다. 결과는 기존 랭킹 순서대로 표시하며 이미 초안에 포함된 `content_id`는 선택할 수 없게 한다. 사용자가 한 장소를 선택하면 타임라인 끝에 추가하고 패널을 닫는다.

랭킹 결과의 다음 페이지를 조회할 수 있다. 빈 결과와 요청 실패는 패널 안에서 안내하며 현재 초안은 유지한다.

### 저장

유효한 초안에서 제목과 4~20자의 비밀번호를 입력해 `POST /api/courses`를 호출한다. 요청은 다음 형식을 사용한다.

```json
{
  "title": "마포에서 천천히 걷는 하루",
  "password": "1234",
  "location_content_ids": ["1", "2", "3"]
}
```

성공하면 응답의 `public_id`로 `/courses/{public_id}`에 이동한다. 비밀번호는 요청 이후 입력란을 비우며 브라우저 저장소, URL, 장기 상태에 보관하지 않는다.

## 공개 코스 조회·수정·삭제

`/courses/{public_id}`는 비밀번호 없이 `GET /api/courses/{public_id}`를 호출한다. 제목, 생성·수정 시각, 방문 순서와 장소 정보, 링크 복사 동작을 표시한다. 거리 관련 응답 필드는 표시하지 않는다.

`수정`을 누르면 현재 제목과 장소 목록을 편집 초안으로 복사한다. 생성 화면과 동일한 순서 변경·삭제·장소 추가 규칙을 사용한다. 저장 시 비밀번호를 입력하고 새 제목과 전체 `location_content_ids`를 `PUT /api/courses/{public_id}`로 보낸다. 실패하면 서버와 화면의 기존 코스를 유지하며 편집 초안도 잃지 않는다. 성공하면 갱신된 응답으로 조회 화면을 다시 그린다.

`삭제`는 별도 확인 대화상자에서 비밀번호를 입력한 뒤 `DELETE /api/courses/{public_id}`를 호출한다. 성공하면 `/courses`로 이동한다. 비밀번호는 수정·삭제 요청에만 사용하고 어느 상태에도 보존하지 않는다.

링크 복사는 현재 출처와 공개 경로를 합친 전체 URL을 사용한다. Clipboard API를 사용할 수 없으면 사용자가 직접 복사할 수 있는 선택 가능한 입력란을 제공한다.

## 모듈 경계

```text
src/features/courses/
  course-api.js          코스 추천·CRUD 요청
  course-state.js        조건·초안 정규화와 순서 편집 순수 함수
  course-validation.js   추천·저장 입력 검증
  course-view.js         장소 카드와 편집 컨트롤 렌더링
  course-builder-page.js `/courses` 생성 작업대
  course-detail-page.js  `/courses/{public_id}` 조회·수정·삭제
  courses.css            코스 화면 전용 반응형 스타일
```

기존 `ranking-api.js`의 메타데이터와 랭킹 조회 함수를 재사용한다. 코스 API 모듈은 코스 도메인의 요청만 담당하고, DOM을 직접 변경하지 않는다. 순서 변경과 중복·개수 검증은 DOM과 분리된 순수 함수로 작성한다.

## 상태와 데이터 흐름

생성 페이지 상태는 `{ metadata, criteria, suggestionStatus, draft, addPlaceState, saveState }`로 제한한다. 추천을 다시 요청할 때만 현재 초안을 교체하며, 장소 추가 조회와 저장 실패는 초안을 변경하지 않는다.

공개 페이지 상태는 `{ courseStatus, course, mode, editDraft, addPlaceState, mutationState }`로 제한한다. `course`는 마지막으로 서버가 확정한 값이고 `editDraft`는 편집 중 복사본이다. 수정 실패 시 `course`를 바꾸지 않는다.

코스 생성·수정 요청에는 화면의 초안 순서를 그대로 `location_content_ids`로 직렬화한다. 프론트엔드는 추천 순위나 거리를 다시 계산하지 않는다.

## 오류 처리

- `VALIDATION_ERROR`: 해당 입력 가까이에 구체적인 검증 메시지 표시
- `COURSE_NOT_ENOUGH_LOCATIONS`: 조건 패널에 다른 구·카테고리 또는 더 적은 장소 수를 선택하도록 안내
- `PASSWORD_MISMATCH`: 비밀번호 입력 오류를 유지하고 다시 입력하도록 안내
- `COURSE_NOT_FOUND`: 공유 코스를 찾을 수 없는 전체 페이지 상태와 새 코스 만들기 링크 표시
- `LOCATION_NOT_FOUND`: 저장·수정 영역에서 장소 목록을 다시 확인하도록 안내
- 네트워크·기타 오류: 현재 입력과 초안을 유지한 채 같은 동작을 재시도할 수 있게 표시

요청 취소는 오류로 표시하지 않는다. 페이지 이탈 시 진행 중인 요청을 취소하고 이벤트 리스너를 정리한다.

## 시각·접근성 방향

기존 파란색 토큰, 둥근 패널, 강한 제목 체계를 유지해 새 기능이 별도 제품처럼 보이지 않게 한다. 코스만의 시각적 표식은 방문 장소를 세로선으로 잇는 편집 타임라인이다. 이 선은 순서 관계만 표현하며 지도 경로나 거리로 보이는 곡선·수치·아이콘은 사용하지 않는다.

모든 컨트롤은 명시적 레이블을 갖고 키보드로 조작할 수 있어야 한다. 비동기 상태와 검증 오류는 `aria-live`로 전달한다. 포커스는 추천 생성 후 타임라인 제목, 장소 추가 후 새 카드, 저장 후 공유 페이지 제목처럼 결과가 생긴 위치로 이동한다. `prefers-reduced-motion`에서는 전환 효과를 제거한다.

## 테스트

### 단위 테스트

- `/courses`와 32자 공개 ID 경로 해석 및 잘못된 경로 거부
- 공통 내비게이션의 코스 링크
- 추천·생성·조회·수정·삭제 요청의 메서드와 본문
- 카테고리 1~3개, 장소 3~5개, 중복 장소, 제목·비밀번호 검증
- 위·아래 순서 변경, 경계 버튼, 최소·최대 장소 수
- 랭킹 장소 추가 시 기존 장소 제외와 페이지 이동
- API 응답의 거리 필드가 DOM과 접근성 텍스트에 나타나지 않음
- 오류 코드별 안내와 실패 시 초안 보존
- 비밀번호가 URL과 브라우저 저장소에 기록되지 않음

### 브라우저 테스트

- 메뉴에서 코스 페이지 이동
- 조건 선택 → 추천 생성 → 순서 변경 → 장소 추가 → 익명 저장
- 공유 링크 조회와 링크 복사
- 비밀번호 오류 후 수정 재시도 및 성공
- 삭제 확인과 성공 후 코스 생성 페이지 이동
- 후보 부족, 코스 없음, 네트워크 실패 상태
- 360px에서 가로 스크롤 없이 생성·공유·편집 가능

## 완료 기준

- 사용자가 새 코스 탭에서 추천 조건을 선택하고 코스 초안을 받는다.
- 사용자가 서로 다른 장소 3~5개의 순서와 구성을 편집해 익명 저장한다.
- 공개 URL로 누구나 코스를 조회하며 올바른 비밀번호로만 수정·삭제한다.
- 직선거리와 총 거리가 모든 화면 및 접근성 텍스트에서 보이지 않는다.
- 전체 성공·오류 흐름이 백엔드 계약과 일치하고 자동화 테스트가 통과한다.
