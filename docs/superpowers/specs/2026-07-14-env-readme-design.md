# 환경 파일 및 README 정리 설계

## 목표

- `.env.example`의 예시 API 주소를 로컬 `.env`로 옮긴다.
- `.env.example`은 저장소에서 삭제한다.
- 프론트엔드 README는 제품 기능과 사용 흐름만 설명하도록 간결하게 개편한다.

## 환경 파일

- 로컬 `.env`에는 `VITE_API_BASE_URL=http://localhost:8000`을 저장한다.
- `.env`는 기존 `.gitignore` 규칙을 유지해 Git에 포함하지 않는다.
- `.env.example`은 삭제한다.
- 브라우저용 프론트엔드에는 서버 전용 비밀키를 두지 않는다.

## README 구성

프론트엔드 `README.md`는 다음 내용만 담는다.

1. 서비스 한 줄 소개
2. 구·카테고리 기반 장소 랭킹과 지도 탐색
3. 익명 게시판과 댓글
4. 근거 기반 지역 정보 챗봇
5. 사용자가 기능을 이용하는 주요 흐름

실행 명령, 테스트 명령, Netlify 배포 설정 등 개발·운영 안내는 제거한다.

## 변경 범위

- 수정: `README.md`
- 삭제: `.env.example`
- 로컬 생성(Git 제외): `.env`
- 루트 저장소의 기존 `README.md` 삭제 상태 및 다른 미추적 디렉터리는 건드리지 않는다.

## 검증

- `.env`에 예시 API 주소가 정확히 들어 있는지 확인한다.
- `.env.example`이 삭제되었는지 확인한다.
- README에 세 핵심 기능과 주요 사용 흐름이 포함되어 있는지 확인한다.
- `.env`가 Git 변경 목록에 나타나지 않는지 확인한다.
