# 뭐할구 랭킹 지도 설계

## 목적

구와 카테고리로 조회한 랭킹을 지도와 함께 탐색하게 한다. 현재 페이지의 장소 분포를 한눈에 보여주고, 랭킹 목록과 지도 마커 어느 쪽에서든 장소를 선택하면 다른 쪽 선택 상태가 즉시 동기화되어야 한다.

## 범위

### 포함

- Leaflet과 OpenStreetMap 타일을 이용한 API 키 없는 지도
- 현재 랭킹 페이지에서 유효한 좌표를 가진 장소의 전체 마커
- 목록 선택 → 마커 강조, 지도 중심 이동, popup 표시
- 마커 선택 → 목록 강조, 플로팅 패널 내부 자동 스크롤
- 데스크톱 지도 위 플로팅 목록과 모바일 세로 배치
- 좌표 없음, 타일 오류, 새 검색·페이지 이동 처리
- OpenStreetMap attribution

### 제외

- 장소 상세 페이지, 길찾기, 사용자 위치, 반경 검색
- 지도에서 구·카테고리 필터 변경
- 마커 clustering, 서버 기반 지도 검색
- 좌표 보정 또는 geocoding

## 기술 결정

`leaflet` npm 패키지를 사용하고 Leaflet CSS를 애플리케이션 번들에 포함한다. 타일 URL은 OpenStreetMap 표준 타일을 사용하며 attribution을 제거하거나 가리지 않는다. 테스트가 외부 타일 네트워크에 의존하지 않도록 Leaflet 접근은 지도 adapter 안에 격리한다.

```text
src/features/ranking/
  ranking-map.js          # Leaflet 지도와 마커 수명주기
  ranking-map-adapter.js  # Leaflet 생성 API 경계
  ranking-page.js         # 랭킹 결과와 선택 상태 조정
  ranking-view.js         # 선택 가능한 랭킹 항목 DOM
  ranking.css             # 플로팅 목록과 반응형 지도
tests/unit/ranking/
  ranking-map.test.js
tests/e2e/
  ranking-map.spec.js
```

## 공개 인터페이스

### `createRankingMap(options)`

```js
createRankingMap({
  container,
  onSelect,
  adapter,
}) => {
  setItems(items),
  select(contentId, { focus = true }),
  invalidateSize(),
  destroy(),
}
```

- `items`는 기존 OpenAPI의 `RankingItem[]`이다.
- `onSelect(contentId)`는 사용자가 마커를 눌렀을 때 호출한다.
- `setItems`는 기존 마커와 popup을 제거하고 현재 페이지 마커를 다시 만든다.
- `select`는 마커 스타일과 popup을 갱신하고 `focus`가 참이면 지도 중심을 이동한다.
- `destroy`는 마커·타일·지도 이벤트와 Leaflet 인스턴스를 정리한다.

### `renderRankingItems(options)`

```js
renderRankingItems(container, items, {
  selectedId,
  onSelect,
})
```

각 랭킹 항목은 키보드로 선택 가능한 버튼 역할을 가진다. 선택 항목에는 `aria-current="true"`와 시각적 활성 클래스를 적용한다. 좌표가 없는 항목에는 “지도 위치 없음”을 표시하고 선택 이벤트는 발생시키되 지도 중심은 변경하지 않는다.

## 좌표와 선택 규칙

`longitude`와 `latitude`가 모두 유한한 숫자이고 각각 `-180~180`, `-90~90` 범위일 때만 지도 좌표로 사용한다. 프론트엔드는 좌표를 보정하거나 주소로부터 새 좌표를 만들지 않는다.

랭킹 조회가 성공하면 다음 순서로 처리한다.

1. 전체 랭킹 목록을 렌더링한다.
2. 유효한 좌표의 항목으로 마커를 만든다.
3. 마커가 하나면 해당 좌표를 서울 지역 탐색에 적합한 zoom으로 표시한다.
4. 마커가 둘 이상이면 전체 마커가 보이도록 bounds를 맞춘다.
5. 최초에는 특정 항목을 선택하지 않는다.

목록 항목을 누르면 `content_id`를 선택 상태의 단일 기준으로 저장한다. 목록을 다시 렌더링해 활성 상태를 표시하고, 유효 좌표가 있으면 지도 마커를 강조해 중심 이동과 popup을 수행한다. 좌표가 없으면 목록 활성 상태만 바뀌고 “이 장소는 지도 위치 정보가 없습니다.”를 상태 영역에 알린다.

마커를 누르면 같은 `content_id`를 선택하고 플로팅 목록의 해당 항목에 포커스를 강제로 옮기지 않은 채 `scrollIntoView({ block: 'nearest' })`로 패널 안에 보이게 한다. 이 방식은 지도 조작 중 키보드 포커스를 빼앗지 않는다.

## 레이아웃

### 데스크톱

랭킹 결과 영역은 높이 `min(70vh, 720px)`, 최소 `520px`인 지도 캔버스다. 목록 패널은 지도 왼쪽 위·아래에 여백을 두고 겹치며 폭은 `min(420px, 40%)`다. 흰색에 가까운 불투명 배경, blur, 테두리, 그림자를 사용해 타일 위에서도 텍스트 대비를 유지한다. 패널 헤더와 페이지네이션은 고정하고 목록 부분만 세로 스크롤한다.

### 모바일

viewport 폭 `720px` 이하에서는 겹침을 해제한다. 지도는 필터 아래에서 폭 100%, 높이 `320px`로 표시하고 목록은 지도 아래 일반 문서 흐름에 둔다. 목록 자체 높이를 제한하지 않아 페이지 스크롤을 사용한다. 360px에서도 지도 zoom control, attribution, 목록 텍스트가 겹치지 않아야 한다.

## 지도 스타일과 접근성

기본 마커는 중립 회청색, 선택 마커는 서비스 primary blue와 더 큰 크기로 구분한다. 색상만이 아니라 크기와 popup으로도 선택을 알린다. popup에는 순위와 장소명, 선택적 주소만 표시한다.

목록 항목은 장소명과 순위를 포함한 접근 가능한 이름을 제공한다. 지도 컨테이너에는 “현재 랭킹 장소 지도” label을 둔다. 지도 선택과 좌표 없음 안내는 `aria-live` 상태 영역에서 텍스트로도 전달한다. `prefers-reduced-motion`이면 지도 이동 animation을 사용하지 않는다.

## 오류와 복구

- 좌표가 하나도 없음: 지도 대신 “현재 결과에는 표시할 위치 정보가 없습니다.”를 렌더링하고 목록은 그대로 유지한다.
- 일부 좌표 없음: 해당 항목만 “지도 위치 없음”을 표시하고 다른 마커는 정상 렌더링한다.
- 타일 오류: 랭킹 목록과 마커 상태를 유지하고 지도 위에 “지도를 불러오지 못했습니다.”와 재시도 버튼을 표시한다.
- 랭킹 API 오류: 기존 랭킹 오류 UI를 사용하며 지도와 플로팅 목록을 비운다.
- 새 검색·페이지 이동: 이전 지도 인스턴스의 이벤트와 레이어를 제거한 뒤 새 결과를 적용한다.

## 테스트

### 단위·DOM

- 좌표 유효성 범위와 `null`, 문자열, `NaN` 제외
- 마커 0개·1개·복수의 empty, setView, fitBounds 동작
- 목록 선택 시 active marker와 popup
- 마커 선택 시 목록 활성화와 `scrollIntoView`
- 좌표 없는 목록 선택 시 지도 불변과 텍스트 안내
- `setItems` 재호출과 `destroy` 시 레이어·이벤트 정리
- 선택 항목의 `aria-current`와 키보드 활성화

### Playwright

- 구·카테고리 조회 후 좌표가 있는 모든 랭킹 마커 표시
- 목록 클릭 후 선택 마커와 popup, 지도 중심 갱신
- 마커 클릭 후 플로팅 목록 항목 강조
- 페이지 이동 후 이전 마커 제거와 새 마커 표시
- 일부·전체 좌표 없음 안내
- 1280×800 플로팅 목록과 360×800 세로 배치·가로 overflow 부재

E2E에서는 타일 요청을 mock하거나 차단해 외부 네트워크 상태가 테스트 결과를 바꾸지 않게 한다.

## 완료 기준

- 현재 페이지의 유효 좌표 장소가 모두 지도에 보인다.
- 목록과 마커 선택이 `content_id` 기준으로 양방향 동기화된다.
- 좌표가 없는 장소도 랭킹에서 사라지지 않는다.
- 새 검색과 페이지 이동 후 이전 지도 상태가 남지 않는다.
- 데스크톱 C안 플로팅 목록과 모바일 세로 배치가 각각 동작한다.
- OpenStreetMap attribution이 항상 보인다.
- 단위·DOM, Playwright, 프로덕션 빌드가 통과한다.
