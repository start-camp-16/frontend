# 뭐할구 프론트엔드 PRD

## 1. 문서 정보

| 항목 | 내용 |
|---|---|
| 제품명 | 뭐할구 |
| 대상 저장소 | 프론트엔드 전용 저장소 |
| 기술 스택 | Vite, Vanilla JavaScript, HTML, CSS |
| 배포 | Netlify |
| API 계약 | `shared/openapi.yaml` |
| 목표 | 서울 구별·카테고리별 장소 목록, 익명 커뮤니티, 지역 정보 챗봇을 제공하는 MVP |

## 2. 제품 원칙

- 로그인 없이 모든 핵심 기능을 사용할 수 있어야 한다.
- 모바일과 데스크톱에서 동일한 기능을 제공한다.
- 프론트엔드는 데이터 순위나 권한을 자체 판단하지 않고 백엔드 응답을 표시한다.
- 비밀번호는 수정·삭제 요청에만 사용하고 브라우저 저장소나 화면 상태에 장기 보관하지 않는다.
- 모든 API 요청·응답 필드는 `shared/openapi.yaml`을 기준으로 한다.

## 3. 사용자 목표

1. 사용자는 서울의 구와 카테고리를 골라 장소 목록을 확인할 수 있다.
2. 사용자는 태그별 게시글을 읽고 제목·본문으로 검색할 수 있다.
3. 사용자는 비밀번호를 이용해 익명 게시글과 댓글을 작성·수정·삭제할 수 있다.
4. 사용자는 챗봇에 서울 장소나 커뮤니티 정보를 질문하고 근거가 포함된 답을 받을 수 있다.

## 4. 정보 구조와 라우트

| 라우트 | 화면 | 주요 기능 |
|---|---|---|
| `/` | 홈/랭킹 | 구·카테고리 선택, 랭킹 목록, 페이지네이션 |
| `/posts` | 게시판 | 태그 탭, 검색, 게시글 목록, 글쓰기 이동 |
| `/posts/new` | 게시글 작성 | 태그·제목·본문·비밀번호 입력 |
| `/posts/:id` | 게시글 상세 | 게시글 표시·수정·삭제, 댓글 CRUD |
| `/posts/:id/edit` | 게시글 수정 | 태그·제목·본문·비밀번호 입력 |

Vite 개발 서버와 Netlify에서 새로고침 시 SPA 진입점으로 연결되도록 fallback 설정을 포함한다.

## 5. 공통 레이아웃

- 상단: 서비스명, 홈, 게시판 이동
- 본문: 현재 라우트의 화면
- 우측 하단: 챗봇 플로팅 버튼
- 모바일 챗봇: 전체 화면 오버레이
- 공통 상태: 로딩, 빈 결과, 오류 안내, 재시도
- 하단: 한국관광공사 TourAPI 데이터 출처와 공공누리 제3유형 표시

## 6. 기능 요구사항

### FE-RANK-01 메타데이터 로딩

- 앱 진입 시 카테고리와 구 목록을 API에서 가져온다.
- 카테고리는 관광지, 레포츠, 문화시설, 쇼핑, 숙박, 여행코스, 축제공연행사를 사용한다.
- 메타데이터 로딩 실패 시 재시도 버튼을 표시한다.

### FE-RANK-02 랭킹 목록

- 구와 카테고리를 모두 선택한 후 랭킹을 요청한다.
- 각 항목에 순위, 장소명, 주소, 대표 이미지가 있으면 이미지, 전화번호가 있으면 전화번호를 표시한다.
- API가 반환한 `rank`를 그대로 사용한다.
- 페이지당 기본 20개를 표시한다.
- 이미지가 없거나 로드되지 않으면 공통 대체 이미지를 표시한다.
- 결과가 없으면 선택 조건에 해당하는 장소가 없다는 메시지를 표시한다.

### FE-POST-01 게시글 목록·검색

- 게시글은 최신 작성순으로 표시한다.
- 상단 탭은 `전체`, `관광`, `맛집`, `문화`, `행사`, `숙박`, `쇼핑`, `자유` 순서다.
- 태그 탭과 검색어를 동시에 적용할 수 있다.
- 검색 대상은 제목과 본문이며 검색 버튼 또는 Enter로 실행한다.
- 검색어를 비우면 선택한 태그의 전체 목록으로 돌아간다.
- 목록에는 태그, 제목, 작성일을 표시한다.

### FE-POST-02 게시글 작성

- 필수 입력은 태그 1개, 제목, 본문, 비밀번호다.
- 제목 1~100자, 본문 1~5,000자, 비밀번호 4~20자를 프론트에서 우선 검증한다.
- 성공하면 생성된 게시글 상세로 이동한다.
- 실패하면 서버 오류 메시지를 입력 폼 주변에 표시하고 입력값을 유지한다.

### FE-POST-03 게시글 상세·수정·삭제

- 상세 화면에 태그, 제목, 본문, 작성·수정 시각을 표시한다.
- 수정 시 기존 비밀번호와 변경할 태그·제목·본문을 전송한다.
- 삭제 전 확인창과 비밀번호 입력창을 표시한다.
- `PASSWORD_MISMATCH`이면 비밀번호가 일치하지 않는다고 안내한다.
- 삭제 성공 후 게시판 목록으로 이동한다.

### FE-COMMENT-01 댓글 목록·작성

- 게시글 상세 진입 시 댓글 목록을 가져온다.
- 댓글 입력은 본문과 댓글용 비밀번호를 요구한다.
- 본문 1~1,000자, 비밀번호 4~20자를 검증한다.
- 작성 성공 후 댓글 목록을 다시 가져온다.

### FE-COMMENT-02 댓글 수정·삭제

- 각 댓글에 수정·삭제 버튼을 제공한다.
- 수정 시 본문과 해당 댓글의 비밀번호를 전송한다.
- 삭제 시 확인창에서 해당 댓글의 비밀번호를 입력받는다.
- 작업 성공 후 댓글 목록을 다시 가져온다.

### FE-CHAT-01 챗봇

- 모든 화면에서 플로팅 버튼으로 열고 닫을 수 있다.
- 현재 브라우저 세션 동안 대화 기록을 메모리에 유지한다.
- 사용자 메시지는 1~1,000자로 제한한다.
- 전송 중에는 중복 전송을 막고 로딩 상태를 표시한다.
- 답변 아래에 근거 장소 또는 게시글 링크·정보를 표시한다.
- `CHAT_RATE_LIMITED`이면 잠시 후 다시 시도하도록 안내한다.
- `CHAT_PROVIDER_ERROR`이면 외부 서비스 연결 실패와 재시도 방법을 안내한다.

## 7. API 연동 원칙

- API 기본 URL은 `VITE_API_BASE_URL` 환경변수로 관리한다.
- 모든 요청은 `/api` 경로의 공유 명세를 따른다.
- 성공 여부를 HTTP 상태 코드로 판단한다.
- 공통 오류의 `message`를 기본 안내로 사용하되 비밀번호, 내부 예외, 환경변수를 화면에 표시하지 않는다.
- 요청 취소 또는 화면 이동 후 도착한 이전 응답이 현재 화면을 덮어쓰지 않게 처리한다.

## 8. 상태 관리

프레임워크 없이 다음 모듈 단위의 메모리 상태만 사용한다.

- `router`: 현재 경로와 라우트 파라미터
- `api`: HTTP 요청과 공통 오류 변환
- `ranking`: 선택 구·카테고리·페이지
- `board`: 태그·검색어·페이지
- `chat`: 열림 상태와 현재 세션 대화 기록

새로고침 후 유지해야 하는 인증·사용자 상태는 없다. 검색어, 태그, 페이지는 URL 쿼리에 반영해 뒤로 가기 동작을 지원한다.

## 9. 접근성·반응형

- 모든 입력에는 연결된 label을 제공한다.
- 키보드만으로 탭, 폼, 모달, 챗봇을 사용할 수 있어야 한다.
- 모달이 열리면 포커스를 내부로 이동하고 닫을 때 원래 버튼으로 돌린다.
- 모바일 기준 폭 360px에서 가로 스크롤 없이 핵심 기능을 사용할 수 있어야 한다.
- 로딩과 오류 상태는 색상 외 텍스트로도 구분한다.

## 10. MVP 제외 범위

- 회원가입·로그인·관리자
- 조회수·좋아요·북마크·실시간 알림
- 댓글 실시간 갱신
- 게시글·댓글 이미지 첨부
- 사용자 정의 태그와 다중 태그
- 고급 검색·자동완성·검색어 저장
- 서버 대화 기록 저장

## 11. 수동 인수 테스트

- 구와 카테고리를 선택해 원본 순서 기반 랭킹을 페이지별로 볼 수 있다.
- 게시글 태그 탭과 검색어를 동시에 적용하고 초기화할 수 있다.
- 게시글을 작성하고 올바른 비밀번호로 수정·삭제할 수 있다.
- 잘못된 비밀번호로 게시글과 댓글을 변경할 수 없다.
- 댓글을 작성·수정·삭제할 수 있다.
- 챗봇 답변과 근거 목록이 표시된다.
- API 및 OpenAI 장애 시 앱이 멈추지 않고 오류와 재시도를 안내한다.
- 모바일에서 게시판과 전체 화면 챗봇을 사용할 수 있다.

## 12. 완료 조건

- `shared/openapi.yaml`의 모든 프론트 사용 API가 연결되어 있다.
- Netlify 배포 환경에서 SPA 라우팅과 API 통신이 동작한다.
- 소스에 API 키나 고정된 운영 서버 주소가 포함되지 않는다.
- 데이터 출처 문구가 모든 주요 화면에서 접근 가능하다.
